CSS Position Property have 4 possible values

  1. Static
  2. Relative
  3. Absolue
  4. Fixed
  5. Sticky

Sticky

Sticky elements are just like relative until they reach the top, then they fix.

Sticky-box

Static

Static is the default position value.

Top, bottom, left, and right will NOT work with static.

A static element will show up exactly where it is in the HTML, left to right, top to bottom... it CANNOT be moved.

Static Box

Relative

A relative positon is the same as static, EXCEPT it will honor top, left, bottom, right.

If you move it, it will sit on top of other stuff on the page, and the page will think it's still where it used to be.

Relative-box

Fixed

A fixed position element will remain, relative to the viewport (the screen) even if the user scrolls.

Top, bottom, left, and right will work just fine.

Un like Relative, it will not reserve its old space. I.e., anything with static or relative will not know it's there.

Fixed-box

Absolute

Absolute elements work off of the grid system. Just like fixed, they are ignored by EVERYONE.

They honor z-index.

Like fixed, they do not leave gaps in the page.

The top/left/bottom/right will be relative to the FIRST parent that has a position set.
